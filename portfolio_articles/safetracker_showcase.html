1. Problem Statement & Scope

Building geolocation trackers is deceptively simple until the device encounters real-world conditions. High battery consumption and aggressive OS power management (like Android Doze mode or iOS background execution limits) often suspend tracking tasks, leading to jagged, missing coordinate paths.

Furthermore, users frequently move through areas with weak cellular networks. Standard apps fail to cache coordinates during connection drops, leading to data loss.

Project Goal: Build a battery-efficient, store-compliant, and consent-first GPS tracking app in React Native using a persistent background task, coupled with a Node.js server that broadcasts coordinates live to Leaflet.js dashboards.

2. System Architecture & Queue Dataflow

The platform is split into a tracking client app and a monitoring server. It utilizes an **Offline-First buffering architecture**:

Mobile-to-Server Data Pipeline

Core Engineering Layers:

  • Background Location Task: Configures Expo's `TaskManager` and `Location` library to fetch coordinates every 10 seconds or 5 meters moved, running as a foreground service with a persistent notification bar to avoid OS suspension.
  • Local Cache Queue: Stores raw location objects in device memory (`AsyncStorage`) during offline gaps. Captures up to 2,000 track-points (approx. 5.5 hours) and uploads them in a single batch post once connection is restored.
  • Hardware Fingerprinting: Resolves native device IDs using hardware strings (`androidId` / `identifierForVendor`) to uniquely identify devices on the dashboard without username sign-ups.
  • Leaflet.js Mapping Engine: Render paths dynamically on a dark-themed grid, drawing accuracy boundaries and historical paths using SVG lines.

3. Technology Stack Selection

React Native & Expo Enables cross-platform iOS and Android compilation. Using Expo's `Location` wrappers ensures proper platform-specific background hooks are executed safely.
AsyncStorage Queue Chosen over SQLite for the offline buffer queue. AsyncStorage handles fast key-value serialization quickly, making it ideal for caching coordinate arrays under low-memory footprints.
Leaflet.js Map Framework Lightweight canvas-based maps that can handle rendering thousands of coordinate trace lines without browser memory leaks.
Node.js WebSocket (ws) Provides dynamic coordinate streams, updating marker positions instantly in the browser UI without long-polling.

4. Code Deep Dive: Background Task Setup

Below is the React Native code initializing the background location task and queueing coordinates offline: 

// React Native Expo Background Task registration
import * as TaskManager from 'expo-task-manager';
import * as Location from 'expo-location';
import AsyncStorage from '@react-native-async-storage/async-storage';

const LOCATION_TASK_NAME = 'background-location-task';

TaskManager.defineTask(LOCATION_TASK_NAME, async ({ data, error }) => {
    if (error) return;
    if (data) {
        const { locations } = data;
        const newCoords = locations.map(loc => ({
            latitude: loc.coords.latitude,
            longitude: loc.coords.longitude,
            timestamp: loc.timestamp
        }));
        
        // Load existing queue
        const rawQueue = await AsyncStorage.getItem('gps_queue') || '[]';
        const queue = JSON.parse(rawQueue);
        queue.push(...newCoords);
        
        // Save back to local storage
        await AsyncStorage.setItem('gps_queue', JSON.stringify(queue));
    }
});

5. Privacy & Ethical Compliance

To comply with Android and iOS app store guidelines, the application implements:

  • Explicit Consent Modal: Displays an unavoidable opt-in window before initializing background location settings.
  • Foreground Notification: Displays a persistent Android status notification (`📍 SafeTracker Active`) while the task runs.
  • One-Tap Kill Switch: Users can completely terminate the background task and clear local buffers directly from the app interface.
