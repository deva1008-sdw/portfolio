1. Problem Statement & Scope

Traditional security scanners (e.g., standard dependency checkers) parse flat lockfiles to identify vulnerabilities (CVEs). While useful, this approach has a major blindspot: it fails to analyze the code execution context.

For example, an application might import a vulnerable library, but if the specific vulnerable function is never invoked in the code, the risk is minimal. Conversely, complex injection chains across files are completely missed by flat-file scanners.

Project Goal: Create a "Code Digital Twin" platform that ingests raw code repositories, parses the code syntax trees down to functions and package imports, maps their structural relationships in a Graph Database, and queries security feeds to map actual call-path risks using LLMs.

2. Architecture & Data Engineering

The platform utilizes a distributed, asynchronous parsing model to construct the graph ledger. Below is the structural layout of the ingestion pipeline:

Code-to-Graph Ingestion Pipeline

Core Component Architecture:

  • Ingestion Engine (tree-sitter): Multi-language parsing worker that recursively traverses files to identify namespaces, imports, function heads, call expressions, and class parameters.
  • Relational DB (Neo4j Aura Cloud): Graph storage layer that correlates codebase entities as visual nodes and maps dependencies dynamically.
  • Audit Integration (OSV.dev API): Standard REST calls querying active dependencies against open source security logs (CVE databases).
  • Auditor LLM (Gemini 1.5 Flash API): Receives target sub-graphs and generates contextual threat assessments based on codebase variables.

3. Technology Stack Choice

Neo4j Graph Database Chosen over relational SQL because mapping multi-level dependency import loops and code call paths requires recursive queries that are highly performant in Cypher.
tree-sitter (Python wrappers) Utilized instead of standard Regex patterns because tree-sitter generates exact, concrete syntax trees (ASTs), ensuring we compile true code blocks.
FastAPI (Python Backend) Provides high-performance asynchronous REST endpoints to handle large codebase uploads and parse operations concurrently.
Google Gemini 1.5 Flash Leveraged for its large context window, enabling the server to send complete sub-graphs of file code straight to the prompt.

4. Technical Deep Dive: AST to Graph

When a codebase is uploaded, the parser reads each file. The Abstract Syntax Tree (AST) generated by tree-sitter is traversed to extract relationships: 

// Cypher schema mapping files to functions and dependency call chains
CREATE CONSTRAINT FOR (f:File) REQUIRE f.path IS UNIQUE;
CREATE CONSTRAINT FOR (fn:Function) REQUIRE fn.name IS UNIQUE;

// Relationship linkage query
MERGE (f:File {path: $filePath})
MERGE (fn:Function {name: $functionName, startLine: $start, endLine: $end})
MERGE (f)-[:CONTAINS]->(fn)
WITH fn
UNWIND $calledFunctions AS targetName
MERGE (t:Function {name: targetName})
MERGE (fn)-[:CALLS]->(t)

Contextual AI Risk Assessment Prompt:

Once vulnerabilities are mapped to specific nodes, the platform queries the Gemini API. Rather than just highlighting a CVE, we feed Gemini the exact call-path sub-graph. The prompt forces a structured JSON output: 

{
  "cve_id": "CVE-2023-XXXX",
  "vulnerable_dependency": "requests",
  "call_path_found": "main.py -> auth.py -> requests.post()",
  "execution_guaranteed": true,
  "risk_score_adjusted": 8.5,
  "remediation_suggestion": "Sanitize target parameters in auth.py:L45 before calling requests.post()"
}

5. Core Features & Business Impact

Digital Twin Code Visualizer Translates thousands of text files into an interactive, visual code-relationship map.
Dead-Code Vulnerability Filter Eliminates false alarms by identifying if a vulnerable package is never called by the application runtime.
Automatic CVE Matching Cross-references dependency package strings with the OSV database to detect compromised versions automatically.
Contextual Remediation Logs Provides exact code-level refactoring advice based on execution context rather than generic package upgrade scripts.
