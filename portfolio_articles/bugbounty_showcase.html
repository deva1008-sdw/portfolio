1. Problem Statement & Scope

Security reconnaissance (recon) is the foundation of any vulnerability audit. However, running discovery tools like domain enumerators, port scanners, and link spiders is highly repetitive and slow.

Off-the-shelf automation scripts run sequentially, meaning that if one tool fails, the pipeline freezes. Furthermore, the outputs of these tools are massive text logs containing 99% noise. Analysts spend hours filtering out dead subdomains, redundant URLs, and standard templates to find real injection points.

Project Goal: Build a high-speed, distributed backend that orchestrates leading offensive binaries (`subfinder`, `httpx`, `katana`) inside background queues, parses outputs structured into a relational database, and routes logs to LLMs to pinpoint attack vectors automatically.

2. System Architecture & Task Queue

The backend is built around a non-blocking asynchronous pipeline. Heavy command execution is deferred to Celery worker pools, keeping the FastAPI REST interface extremely responsive:

Asynchronous Recon Ingestion Workflow

Functional Data Stages:

  • Endpoint Trigger: User registers a root domain (e.g. `domain.com`) via REST POST.
  • Task Distribution: Celery passes host discovery parameters to a worker thread via Redis Broker.
  • Utility Execution: Worker spawns shell sub-processes running `subfinder` and `httpx`, collecting responsive subdomains and checking port availability.
  • Crawler Crawl: Katana spiders directories and endpoints, extracting form URLs, input parameters, and JavaScript scripts.
  • LLM Anomaly Filter: Ingested URLs and header logs are audited by Gemini/OpenAI API prompts to flag credentials files or parameter exposures.

3. Technology Stack Selection

FastAPI (Python) Chosen for its native async support. It manages WebSocket streaming of scan logs and target creations at high speeds.
Celery & Redis Decouples heavy networking tasks from the API. Celery worker scaling keeps API responses quick even during large scans.
PostgreSQL DB Relational database structure linking domains to subdomains, ports, URLs, and AI-triage results with integrity.
Go-Based CLI Utilities Wrapped Go binaries (ProjectDiscovery tools) are highly optimized for network polling, outperforming Python socket loops.

4. Code Deep Dive: Binary Execution Wrapper

Below is the Python wrapper code used to capture Go-binary output, parsing JSON logs straight into database rows asynchronously: 

# Worker subprocess executor that executes Katana spider binary
import subprocess
import json

def run_spider_worker(target_id: str, domain: str):
    # Command array executing crawl
    cmd = [
        "katana",
        "-u", domain,
        "-jc",          # Crawl JS code
        "-kf", "all",   # Keep all fields
        "-silent",
        "-json"
    ]
    
    # Spawn subprocess capturing stdout
    process = subprocess.Popen(cmd, stdout=subprocess.PIPE, stderr=subprocess.PIPE, text=True)
    
    db = SessionLocal()
    while True:
        line = process.stdout.readline()
        if not line:
            break
            
        data = json.loads(line)
        crawled_url = data.get("request", {}).get("endpoint")
        
        # Save to database
        db_url = CrawledURL(target_id=target_id, url=crawled_url)
        db.add(db_url)
        db.commit()
    db.close()
